01 — Fundamentos

O que é o Arduino, de fato

O Arduino não é um único componente — é uma plataforma completa que combina uma placa de circuito (hardware), um microcontrolador programável e um ambiente de desenvolvimento (IDE) gratuito. No GAIA Robótica, é a peça que conecta código a movimento físico: sensores, motores, LEDs e módulos de comunicação passam todos pelo mesmo cérebro central.

O modelo mais usado nos nossos projetos — o Arduino Uno R3 — utiliza o microcontrolador ATmega328P, um chip de arquitetura RISC de 8 bits que executa instruções em ciclos de clock únicos, o que garante respostas em tempo real suficientes para controle de motores e leitura de sensores.

Por que ele domina a robótica educacional

Custo baixo, comunidade enorme, e uma linguagem baseada em C/C++ que é praticamente idêntica ao que se usa em microcontroladores industriais. Aprender Arduino é o primeiro passo real para sistemas embarcados profissionais.

Entradas x Saídas Digitais

Pinos digitais operam em dois estados: HIGH (5V) ou LOW (0V). São usados para ligar/desligar componentes — relés, LEDs, sinais lógicos. Pinos marcados com ~ suportam PWM, permitindo simular um valor analógico de saída através de pulsos rápidos.

Entradas Analógicas

Os pinos A0 a A5 leem tensões contínuas entre 0V e 5V, convertendo-as em um valor digital de 0 a 1023 através de um conversor Analógico-Digital (ADC) de 10 bits interno ao chip. É assim que sensores como potenciômetros, LDRs e sensores IR analógicos são lidos.

03 — Programação

Estrutura do código

Todo programa Arduino (chamado de sketch) segue uma estrutura fixa de duas funções obrigatórias.

C/C++ — Arduino blink.ino 
// Estrutura mínima de um sketch Arduino

const int LED = 13;

void setup() {
  // Executa uma única vez, ao ligar a placa
  pinMode(LED, OUTPUT);
}

void loop() {
  // Executa em repetição infinita
  digitalWrite(LED, HIGH);
  delay(1000);
  digitalWrite(LED, LOW);
  delay(1000);
}

Modulação por Largura de Pulso (PWM)

Como o Arduino não tem saída analógica de verdade, ele simula uma usando PWM: liga e desliga o pino em alta frequência, variando a proporção de tempo em nível alto (duty cycle). O resultado prático é uma tensão média equivalente — usada para controlar brilho de LEDs e velocidade de motores.

C/C++ — Arduino pwm_motor.ino 
// Controle de velocidade via PWM (0 a 255)

const int PIN_MOTOR = 9;  // pino com suporte PWM (~)

void setup() {
  pinMode(PIN_MOTOR, OUTPUT);
}

void loop() {
  analogWrite(PIN_MOTOR, 180);  // ~70% de potência
}

Boas práticas de código

Fazer

Nomear constantes com MAIÚSCULAS e comentar o motivo de cada pino escolhido — facilita revisão em equipe.

Evitar

Usar números "mágicos" direto no código sem explicação (ex: digitalWrite(7, HIGH) sem contexto).

Fazer

Testar sensores isoladamente antes de integrar ao sistema completo — isola a origem de bugs.

Evitar

Alimentar motores diretamente pelos pinos do Arduino — a corrente máxima suportada é de cerca de 40mA por pino.

05 — Indo Além

Depois do Arduino Uno

Uma vez confortável com o Uno, o próximo passo natural no GAIA é o ESP32 — um microcontrolador com Wi-Fi e Bluetooth integrados, usado em projetos que precisam de comunicação sem fio, como telemetria de drones e controle remoto de robôs.

Para aprofundar no controle de motores com precisão, veja o artigo sobre Controle PID — ele explica como transformar o PWM básico visto aqui em um sistema de controle de malha fechada.
